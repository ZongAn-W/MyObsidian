# 第六章 关系数据理论：重要知识点总结

> 来源：《数据库系统概论（第6版）》第6章。重点掌握函数依赖、码、范式、规范化、Armstrong 公理系统、最小依赖集、保持函数依赖分解和无损连接分解。

## 1. 本章学习主线

关系数据理论主要用于回答数据库逻辑设计中的两个问题：

1. 怎样判断一个关系模式是否“好”？
2. 如果关系模式“不好”，怎样把它分解成更好的关系模式？

本章核心思想：

```text
关系模式不好
    -> 数据冗余、更新异常、插入异常、删除异常
分析原因
    -> 函数依赖、多值依赖等数据依赖不合理
解决方法
    -> 模式分解，使关系模式达到更高范式
```

## 2. 问题的提出

### 2.1 关系模式的形式化表示

关系模式可形式化表示为：

```text
R(U, D, DOM, F)
```

含义：

| 符号 | 含义 |
| --- | --- |
| R | 关系名 |
| U | 属性集合 |
| D | 属性来自的域 |
| DOM | 属性到域的映射 |
| F | 属性之间的数据依赖集合 |

本章通常简化为：

```text
R(U, F)
```

因为规范化理论主要关心属性集合 `U` 和数据依赖集合 `F`。

### 2.2 不好的关系模式会产生的问题

若把过多语义放进一个大关系中，可能出现：

| 问题 | 含义 |
| --- | --- |
| 数据冗余 | 同一事实重复存储 |
| 更新异常 | 修改一个事实时需要改多处，容易不一致 |
| 插入异常 | 某些信息因缺少其他信息而无法插入 |
| 删除异常 | 删除某些元组时，把本不该删除的信息也删掉 |

例子：若把学生、学院、院长、选课、成绩都放在一个关系 `Student(Sno, School, Mname, Cno, Grade)` 中：

- 学院院长姓名会在很多学生记录中重复。
- 学院更换院长时，需要修改大量元组。
- 新学院暂时没有学生时，学院和院长信息无法插入。
- 某学院学生全部毕业后，删除学生记录会导致学院信息丢失。

解决思路：把一个“混杂”的关系分解为多个语义单一的关系。

- 函数依赖：functional dependency，FD。
- 多值依赖：multivalued dependency，MVD。

## 8. Armstrong 公理系统

Armstrong 公理系统用于从已知函数依赖集合 `F` 推导出其他函数依赖。

### 8.1 函数依赖的逻辑蕴涵

若 `F` 中的函数依赖都成立时，`X -> Y` 必然成立，则称 `F` 逻辑蕴涵 `X -> Y`。

### 8.2 三条基本推理规则

| 规则 | 形式 | 含义 |
| --- | --- | --- |
| 自反律 | 若 `Y ⊆ X ⊆ U`，则 `X -> Y` | 属性组能决定自己的子集 |
| 增广律 | 若 `X -> Y`，则 `XZ -> YZ` | 两边同时加属性仍成立 |
| 传递律 | 若 `X -> Y`，`Y -> Z`，则 `X -> Z` | 依赖可传递 |

这三条规则是有效的、完备的。

### 8.3 常用推理规则

由 Armstrong 公理可推出：

| 规则 | 形式 |
| --- | --- |
| 合并规则 | 若 `X -> Y` 且 `X -> Z`，则 `X -> YZ` |
| 伪传递规则 | 若 `X -> Y` 且 `WY -> Z`，则 `XW -> Z` |
| 分解规则 | 若 `X -> Y` 且 `Z ⊆ Y`，则 `X -> Z` |

常用结论：

```text
X -> A1A2...Ak 成立
当且仅当 X -> Ai 对每个 i 都成立
```

## 9. 闭包与最小依赖集

### 9.1 函数依赖集闭包 F+

由函数依赖集 `F` 根据 Armstrong 公理系统能够推导出的所有函数依赖，称为 `F` 的闭包，记作：

```text
F+
```

### 9.2 属性集闭包 X+

设 `X` 是属性集合，`X` 关于 `F` 的闭包记作：

```text
X_F+
```

含义：由 `X` 根据 `F` 能函数确定的全部属性。

用途：

- 判断 `X -> Y` 是否由 `F` 推出：看 `Y ⊆ X_F+`。
- 判断 `X` 是否为码：看 `X_F+ = U`。
- 求候选码。

### 9.3 求属性闭包的基本步骤

```text
1. 初始令 X+ = X
2. 扫描 F 中的每个依赖 A -> B
3. 如果 A ⊆ X+，则把 B 加入 X+
4. 重复扫描，直到 X+ 不再变化
```

若最终 `X+ = U`，则 `X` 是超码；若再满足最小性，则 `X` 是候选码。

### 9.4 函数依赖集等价与覆盖

若：

```text
F+ = G+
```

则称 `F` 和 `G` 等价，或 `F` 覆盖 `G` 且 `G` 覆盖 `F`。

判断依赖集等价的常用方法：

```text
逐个检查 F 中的依赖是否能由 G 推出
再逐个检查 G 中的依赖是否能由 F 推出
```

### 9.5 最小依赖集

若函数依赖集 `F` 满足：

1. 每个函数依赖右部只含一个属性。
2. 不存在多余的函数依赖。
3. 每个函数依赖左部不存在多余属性。

则 `F` 是一个最小依赖集，也称最小覆盖。

求最小依赖集的常用步骤：

```text
1. 把右部分解成单属性
2. 删除冗余依赖
3. 删除左部冗余属性
```

注意：最小依赖集不一定唯一。

## 10. 模式分解

关系模式分解是把一个关系模式 `R(U, F)` 分解成若干个较小的关系模式。

好的分解应尽量满足：

1. 无损连接。
2. 保持函数依赖。
3. 分解后的关系达到较高范式。

这三个目标不一定总能同时完美满足。

## 11. 保持函数依赖的模式分解

### 11.1 保持函数依赖的含义

若分解后各子关系上的依赖合起来，与原来的依赖集等价，则称该分解保持函数依赖。

直观理解：

```text
原来能检查的约束，分解后仍能在各子表中直接检查。
```

若不保持函数依赖，某些约束可能必须通过连接多个表后才能检查，代价较高。

### 11.2 合成法

合成法可得到一个保持函数依赖、达到 3NF 的分解。

基本思路：

```text
1. 先求 F 的最小依赖集
2. 按相同左部把依赖分组
3. 每组依赖涉及的属性构成一个关系模式
4. 若有属性未出现在任何依赖中，单独构成关系
5. 去掉被其他关系模式包含的冗余关系
```

结论：

- 合成法保持函数依赖。
- 分解后的关系模式属于 3NF。
- 不一定达到 BCNF。

## 12. 无损连接的模式分解

### 12.1 无损连接的含义

分解后，如果通过自然连接能够恢复出原关系，不产生多余元组，也不丢失元组，则称该分解具有无损连接性。

记忆：

```text
无损连接 = 分解后再连接，能恢复原关系
```

注意：

- 无损连接关注“数据是否能恢复”。
- 保持函数依赖关注“约束是否能保留”。
- 二者是不同概念。

### 12.2 二元分解的无损连接判定

设关系 `R` 分解为：

```text
R1(U1), R2(U2)
```

若满足：

```text
(U1 ∩ U2) -> (U1 - U2)
```

或：

```text
(U1 ∩ U2) -> (U2 - U1)
```

则该分解具有无损连接性。

通俗理解：

```text
两个子关系的公共属性必须能决定其中一边的非公共属性。
```

### 12.3 分解算法的取舍

| 算法/目标 | 结果特点 |
| --- | --- |
| 合成法 | 保持函数依赖，达到 3NF，但不一定无损或不一定 BCNF |
| 3NF 无损且保持依赖分解 | 既无损连接，又保持函数依赖，达到 3NF |
| BCNF 分解法 | 可达到 BCNF，通常无损连接，但不一定保持函数依赖 |
| 4NF 分解法 | 在多值依赖条件下达到 4NF，并保持无损连接 |

考试常考：

```text
3NF：通常能同时做到无损连接和保持函数依赖
BCNF：能做到无损连接，但可能不能保持函数依赖
```

## 13. 易混概念辨析

### 13.1 候选码、主码、超码

| 概念 | 是否能唯一标识元组 | 是否最小 | 备注 |
| --- | --- | --- | --- |
| 超码 | 能 | 不一定 | 可含多余属性 |
| 候选码 | 能 | 是 | 最小超码 |
| 主码 | 能 | 是 | 从候选码中选定一个 |

### 13.2 2NF、3NF、BCNF

| 范式 | 核心要求 | 主要消除 |
| --- | --- | --- |
| 2NF | 非主属性完全依赖于码 | 部分依赖 |
| 3NF | 非主属性不传递依赖于码 | 传递依赖 |
| BCNF | 每个决定因素都含有码 | 更严格的决定因素异常 |

### 13.3 函数依赖与多值依赖

| 概念 | 记号 | 含义 |
| --- | --- | --- |
| 函数依赖 | `X -> Y` | 一个 `X` 值对应唯一 `Y` 值 |
| 多值依赖 | `X ->-> Y` | 一个 `X` 值对应一组 `Y` 值，且与其他属性独立 |

函数依赖可看作多值依赖的特殊情况，但规范化判断中二者解决的问题不同。

### 13.4 保持函数依赖与无损连接

| 概念 | 关注点 |
| --- | --- |
| 保持函数依赖 | 约束能否在分解后的关系中保留和检查 |
| 无损连接 | 分解后的关系自然连接能否恢复原关系 |

两个性质相互独立：

- 可能保持函数依赖但不是无损连接。
- 可能无损连接但不保持函数依赖。

## 14. 高频考点

- 不好的关系模式会产生数据冗余、更新异常、插入异常和删除异常。
- 函数依赖 `X -> Y` 的定义和语义判断。
- 平凡函数依赖、非平凡函数依赖。
- 完全函数依赖、部分函数依赖、传递函数依赖。
- 候选码、超码、主码、主属性、非主属性、外码。
- 1NF、2NF、3NF、BCNF、4NF 的定义和包含关系。
- 2NF 消除非主属性对码的部分依赖。
- 3NF 消除非主属性对码的传递依赖。
- BCNF 要求每个决定因素都包含码。
- 多值依赖和 4NF 的含义。
- Armstrong 公理系统：自反律、增广律、传递律。
- 合并规则、伪传递规则、分解规则。
- 属性闭包 `X+` 的计算。
- 用属性闭包判断函数依赖是否成立、判断码。
- 最小依赖集的三个条件和求法。
- 保持函数依赖分解的含义。
- 无损连接分解的含义。
- 二元分解无损连接的判定条件。
- 3NF、BCNF 分解在“无损连接”和“保持依赖”上的区别。
