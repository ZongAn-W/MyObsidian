规范化的基本思想是：

```text
逐步消除不合适的数据依赖
把一个关系分解成若干个更好的关系
使每个关系只表达一个概念或一个实体间联系
```

各范式解决的问题：

| 范式 | 主要消除的问题 |
| --- | --- |
| 1NF | 属性不可再分 |
| 2NF | 非主属性对码的部分函数依赖 |
| 3NF | 非主属性对码的传递函数依赖 |
| BCNF | 所有属性对非码决定因素的依赖 |
| 4NF | 非平凡且非函数依赖的多值依赖 |

考试常用判断顺序：

```text
先找候选码
再分主属性和非主属性
判断是否有部分依赖
判断是否有传递依赖
判断每个决定因素是否含有码
若涉及多值依赖，再判断 4NF
```
