用户定义完整性是针对具体应用语义提出的数据约束。

常见约束：

- 列值非空：`NOT NULL`
- 列值唯一：`UNIQUE`
- 条件检查：`CHECK`

### 5.1 属性上的约束

非空约束：

```sql
Sname CHAR(20) NOT NULL
```

唯一约束：

```sql
SHname VARCHAR(40) UNIQUE
```

检查约束：

```sql
Ssex CHAR(6) CHECK (Ssex IN ('男', '女'))
```

成绩范围：

```sql
Grade SMALLINT CHECK (Grade BETWEEN 0 AND 100)
```

### 5.2 元组上的约束

元组级约束可以涉及多个属性之间的关系。

例子：

```sql
CREATE TABLE Teacher (
    Eno CHAR(8) PRIMARY KEY,
    Ename VARCHAR(20),
    Sal NUMERIC(7,2),
    Deduct NUMERIC(7,2),
    CHECK (Sal + Deduct >= 3000)
);
```

含义：应发工资 `Sal + Deduct` 不低于 3000。

### 5.3 用户定义完整性的检查

当插入元组或修改属性值时，DBMS 会检查用户定义约束是否满足。

若不满足，操作通常被拒绝执行。
