关系代数是一种抽象的查询语言，用对关系的运算表达查询。

关系代数运算对象是关系，运算结果仍然是关系。

### 8.1 传统集合运算

传统集合运算包括：

- 并
- 差
- 交
- 笛卡尔积

设关系 `R` 和 `S` 具有相同的目 `n`，且对应属性来自同一个域。

#### 并

```text
R ∪ S = { t | t ∈ R 或 t ∈ S }
```

结果仍为 `n` 目关系，由属于 `R` 或属于 `S` 的元组组成。

#### 差

```text
R - S = { t | t ∈ R 且 t ∉ S }
```

结果由属于 `R` 但不属于 `S` 的元组组成。

#### 交

```text
R ∩ S = { t | t ∈ R 且 t ∈ S }
```

交可以用差表示：

```text
R ∩ S = R - (R - S)
```

#### 笛卡尔积

若 `R` 为 `n` 目关系，有 `k1` 个元组；`S` 为 `m` 目关系，有 `k2` 个元组，则：

- `R x S` 是 `n + m` 目关系。
- 元组个数为 `k1 x k2`。

### 8.2 专门的关系运算

专门的关系运算包括：

- 选择
- 投影
- 连接
- 除

#### 选择

选择是从关系中选出满足条件的元组。

记法：

```text
σF(R)
```

含义：从关系 `R` 中选择满足条件 `F` 的元组。

对应 SQL 思维：`WHERE` 过滤行。

#### 投影

投影是从关系中选出若干属性列，并去掉重复元组。

记法：

```text
πA(R)
```

含义：从关系 `R` 中取属性列 `A`。

对应 SQL 思维：`SELECT` 指定列。

#### 连接

连接是从两个关系的笛卡尔积中选出满足条件的元组。

一般形式：

```text
R ⋈ S
```

常见连接：

- 等值连接：连接条件使用 `=`
- 自然连接：特殊的等值连接，会比较同名属性，并在结果中去掉重复属性列

自然连接要点：

- 要求两个关系有同名属性。
- 同名属性值相等的元组才能连接。
- 结果中同名属性只保留一列。

#### 除

除运算用于表达“全部”“至少包含所有”等查询。

直观理解：

```text
R ÷ S
```

返回那些在 `R` 中能够与 `S` 的所有元组都匹配的属性值。

典型题型：

- 查询选修了全部课程的学生。
- 查询供应了某工程所需全部零件的供应商。
- 查询至少选修了某几个指定课程的学生。
