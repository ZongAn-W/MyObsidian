### 9.1 触发器的含义

触发器是由事件触发、自动执行的一段数据库程序。

触发器可以实现更复杂的完整性检查和违约处理。

触发器规则通常表示为：

```text
事件 - 条件 - 动作
```

即：当某个事件发生时，检查条件；若条件成立，则执行动作。

### 9.2 创建触发器

一般格式：

```sql
CREATE TRIGGER <触发器名>
{BEFORE | AFTER} <触发事件> ON <表名>
REFERENCING NEW | OLD AS <变量>
FOR EACH {ROW | STATEMENT}
[WHEN <触发条件>]
<触发动作体>;
```

### 9.3 触发器关键组成

| 组成 | 含义 |
| --- | --- |
| 触发器名 | 标识触发器 |
| 目标表 | 触发器所依附的基本表 |
| 触发事件 | INSERT、DELETE、UPDATE 等 |
| BEFORE / AFTER | 触发时机 |
| FOR EACH ROW | 行级触发器，每影响一行执行一次 |
| FOR EACH STATEMENT | 语句级触发器，一条语句执行一次 |
| WHEN | 触发条件 |
| 触发动作体 | 条件成立时执行的动作 |

### 9.4 NEW 与 OLD

在行级触发器中：

- `NEW` 表示更新或插入后的新值。
- `OLD` 表示更新或删除前的旧值。

注意：语句级触发器通常不能在动作体中使用 `NEW` 或 `OLD` 引用单行值。

### 9.5 触发器的特点

优点：

- 可以实现复杂约束。
- 可以自动处理违约。
- 可以用于审计、安全、业务流程控制等。

风险：

- 触发器动作复杂，不易调试。
- 一个触发器可能激活另一个触发器，形成触发链。
- 设计不当可能造成难以预见的错误。
