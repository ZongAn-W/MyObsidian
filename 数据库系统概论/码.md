
### 5.1 候选码

设 `K` 是 `R(U, F)` 中的属性或属性组。若：

```text
K -> U
```

并且 `K` 的任意真子集都不能决定 `U`，则称 `K` 为 `R` 的候选码。

理解：

```text
候选码 = 能唯一标识一个元组的最小属性组
```

注意：

- `K -> U` 要求 `U` 完全函数依赖于 `K`。
- 候选码可以有多个。
- 若候选码有多个，可以选定一个作为主码。

### 5.2 超码

若 `K -> U`，但 `K` 不一定最小，则 `K` 是超码。

关系：

```text
候选码一定是超码
超码不一定是候选码
```

例子：如果 `Sno` 是候选码，则 `(Sno, Sname)` 也是超码，但不是候选码。

### 5.3 主属性与非主属性

| 概念   | 含义            |
| ---- | ------------- |
| 主属性  | 包含在任一候选码中的属性  |
| 非主属性 | 不包含在任何候选码中的属性 |

范式判断时，经常要看“非主属性”是否对码存在部分依赖或传递依赖。

### 5.4 外码

若 `X` 不是关系模式 `R` 的码，但 `X` 是另一个关系模式的码，则称 `X` 是 `R` 的外码。

外码用于表达关系之间的联系。
