# 第五章 数据库完整性：重要知识点总结

> 来源：《数据库系统概论（第6版）》第5章。重点掌握完整性约束、实体完整性、参照完整性、用户定义完整性、命名约束、断言和触发器。

## 1. 本章学习主线

数据库完整性是指数据库中数据的正确性和相容性。

本章主要回答三个问题：

1. 什么是数据库完整性？
2. 如何用 SQL 定义和检查完整性约束？
3. 违反完整性约束时，DBMS 如何处理？

核心记忆：

```text
完整性 = 正确性 + 相容性
完整性控制 = 定义约束 + 检查约束 + 违约处理
```

## 10. 易混概念辨析

### 10.1 实体完整性、参照完整性、用户定义完整性

| 类型 | 约束对象 | 典型 SQL |
| --- | --- | --- |
| 实体完整性 | 主码 | PRIMARY KEY |
| 参照完整性 | 外码 | FOREIGN KEY REFERENCES |
| 用户定义完整性 | 应用语义 | NOT NULL、UNIQUE、CHECK |

### 10.2 NO ACTION 与 CASCADE

| 策略 | 含义 |
| --- | --- |
| NO ACTION | 违反约束时拒绝执行 |
| CASCADE | 对相关表级联执行相应操作 |

### 10.3 CHECK 与 ASSERTION

| 概念 | 作用范围 |
| --- | --- |
| CHECK | 多用于列级、元组级或表级约束 |
| ASSERTION | 数据库级全局约束，可涉及多个表 |

### 10.4 触发器与普通约束

| 概念 | 特点 |
| --- | --- |
| 普通约束 | 声明式，简单清晰 |
| 触发器 | 过程式，功能强但复杂 |

## 11. 高频考点

- 数据库完整性的定义：正确性和相容性。
- 完整性与安全性的区别。
- DBMS 完整性控制的三类功能。
- 实体完整性用 `PRIMARY KEY` 定义。
- 单属性主码和多属性主码的定义方式。
- 参照完整性用 `FOREIGN KEY ... REFERENCES` 定义。
- 参照完整性违约处理策略：`NO ACTION`、`CASCADE`、`SET NULL`、`SET DEFAULT`。
- 用户定义完整性：`NOT NULL`、`UNIQUE`、`CHECK`。
- 元组级约束可以涉及多个属性。
- `CONSTRAINT` 命名约束的作用。
- 用 `ALTER TABLE` 增加或删除约束。
- 断言 `CREATE ASSERTION` 的作用。
- 触发器的事件、条件、动作。
- `BEFORE` 与 `AFTER` 的区别。
- 行级触发器与语句级触发器的区别。
- `NEW` 和 `OLD` 的含义。
