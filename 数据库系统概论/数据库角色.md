### 7.1 角色的含义

角色是被命名的一组权限的集合。

使用角色可以简化权限管理：

- 先把权限授予角色。
- 再把角色授予用户。
- 用户通过角色获得一组权限。

### 7.2 创建角色

```sql
CREATE ROLE <角色名>;
```

例子：

```sql
CREATE ROLE R1;
```

### 7.3 给角色授权

```sql
GRANT SELECT, UPDATE, INSERT
ON TABLE Student
TO R1;
```

### 7.4 将角色授予用户或其他角色

```sql
GRANT <角色名>
TO <用户或角色>
[WITH ADMIN OPTION];
```

`WITH ADMIN OPTION` 表示获得该角色的用户或角色可以继续把该角色授予别人。

### 7.5 回收角色权限

```sql
REVOKE <权限>
ON <对象类型> <对象名>
FROM <角色名>;
```

角色的优点：

- 降低重复授权工作量。
- 便于按岗位管理权限。
- 便于统一调整一组用户的权限。
