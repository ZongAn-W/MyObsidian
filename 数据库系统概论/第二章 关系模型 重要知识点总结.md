# 第二章 关系模型：重要知识点总结

> 来源：《数据库系统概论（第6版）》第2章。重点围绕关系模型的数据结构、关系操作、完整性约束和关系代数展开。

## 1. 本章学习主线

第二章是关系数据库的理论基础，后续 SQL、数据库完整性、安全性和数据库设计都会反复使用本章概念。

本章主要回答四个问题：

1. 关系模型的数据结构是什么？
2. 关系模型能进行哪些操作？
3. 关系模型必须满足哪些完整性约束？
4. 如何用关系代数表达查询？

核心记忆：

```text
关系模型 = 关系数据结构 + 关系操作 + 关系完整性约束
```

## 2. 关系模型的数据结构

### 2.1 关系模型的基本特点

关系模型的数据结构非常简单，只有一种数据结构：关系。

从用户角度看，关系就是一张二维表；但严格地说，关系是笛卡尔积的一个有意义的子集。

关系模型用统一的“关系”结构表示：

- 实体
- 实体的属性
- 实体之间的联系

这是关系模型区别于层次模型、网状模型的重要特点。

### 2.2 域

域是一组具有相同数据类型的值的集合。

例子：

- 性别域：{男，女}
- 年龄域：0 到 100 之间的整数
- 姓名域：若干字符串

要点：域规定了属性可以取什么值。

### 2.3 笛卡尔积

给定一组域 `D1, D2, ..., Dn`，它们的笛卡尔积为：

```text
D1 x D2 x ... x Dn = { (d1, d2, ..., dn) | di ∈ Di, i = 1, 2, ..., n }
```

其中：

- 每个元素 `(d1, d2, ..., dn)` 称为一个 n 元组。
- 元组中的每个值称为一个分量。
- 若每个域 `Di` 的基数为 `mi`，则笛卡尔积的基数为 `m1 x m2 x ... x mn`。

理解：笛卡尔积是所有可能组合的集合，但其中大部分组合通常没有实际语义。

### 2.4 关系

关系是笛卡尔积的一个有意义的子集。

在关系模型中：

- 关系可以看作一张二维表。
- 表的一行是一个元组。
- 表的一列是一个属性。
- 属性的取值范围来自某个域。

常见术语：

| 术语 | 含义 |
| --- | --- |
| 关系 | 二维表 |
| 元组 | 表中的一行 |
| 属性 | 表中的一列 |
| 域 | 属性的取值范围 |
| 分量 | 元组中的一个属性值 |
| 关系模式 | 对关系结构的描述 |
| 关系数据库 | 若干关系的集合 |

### 2.5 关系的性质

关系作为规范化的二维表，通常具有以下性质：

- 每一列中的分量来自同一个域。
- 不同列可以来自同一个域，但必须用不同属性名区分。
- 列的顺序可以交换。
- 任意两个元组不能完全相同。
- 行的顺序可以交换。
- 每个分量必须是不可再分的数据项，即满足第一范式要求。

注意：关系中的“表”不是普通表格，而是有严格数学约束的二维表。

## 3. 关系模式

### 3.1 关系模式的含义

关系模式是对关系的描述，可以形式化表示为：

```text
R(U, D, DOM, F)
```

其中：

- `R`：关系名。
- `U`：组成该关系的属性名集合。
- `D`：属性来自的域。
- `DOM`：属性向域的映像集合。
- `F`：属性间数据依赖关系集合。

本章通常简记为：

```text
R(U)
```

或：

```text
R(A1, A2, ..., An)
```

例如：

```text
学生(学号, 姓名, 性别, 出生日期, 主修专业)
```

### 3.2 型和值

关系模式是型，关系是值。

可以这样理解：

- 关系模式描述表结构，相对稳定。
- 关系是某一时刻表中的具体数据，会随插入、删除、修改而变化。

类似于程序设计中的“类型”和“变量值”的关系。

## 4. 码相关概念

### 4.1 候选码

若关系中的某一个属性或属性组能够唯一标识一个元组，并且它的真子集不能唯一标识元组，则称该属性或属性组为候选码。

关键词：

- 唯一标识
- 最小属性组

### 4.2 主码

如果一个关系有多个候选码，则选择其中一个作为主码，也称主键。

主码用于唯一标识关系中的元组。

### 4.3 主属性与非主属性

- 主属性：包含在任一候选码中的属性。
- 非主属性：不包含在任何候选码中的属性，也称非码属性。

### 4.4 全码

若关系模式的所有属性共同构成候选码，则称为全码。

### 4.5 外码

设 `F` 是关系 `R` 的一个或一组属性，但不是 `R` 的码；若 `F` 与关系 `S` 的主码 `Ks` 相对应，则称 `F` 是 `R` 的外码。

相关术语：

- `R`：参照关系。
- `S`：被参照关系，也称目标关系。
- 外码和被参照关系的主码必须定义在同一个域或同一组域上。

典型例子：

```text
学生(学号, 姓名, 主修专业)
专业(专业名, 专业编号)
```

若 `学生.主修专业` 参照 `专业.专业名`，则 `主修专业` 是学生关系的外码。

## 5. 关系操作

### 5.1 基本关系操作

关系操作包括查询和更新两大类。

更新操作包括：

- 插入
- 删除
- 修改

查询操作包括：

- 选择
- 投影
- 连接
- 除
- 并
- 差
- 交
- 笛卡尔积

其中，选择、投影、并、差、笛卡尔积是 5 种基本操作，其他操作可以由基本操作定义和导出。

### 5.2 集合操作方式

关系操作的特点是集合操作方式，也称一次一集合方式。

也就是说：

- 操作对象是关系。
- 操作结果仍然是关系。
- 中间结果也是关系。

这与层次模型、网状模型中一次一个记录的操作方式不同。

## 6. 关系数据语言

关系数据语言可以分为三类：

1. 关系代数语言：用关系运算表达查询，如 ISBL。
2. 关系演算语言：用谓词表达查询。
3. 结构化查询语言：SQL，兼具关系代数和关系演算特点。

关系演算又可分为：

- 元组关系演算，如 ALPHA、QUEL。
- 域关系演算，如 QBE。

SQL 的特点：

- 不仅有查询功能。
- 还包含数据定义、数据操纵、数据控制等功能。
- 是集 DQL、DDL、DML、DCL 于一体的关系数据库语言。
- 从 20 世纪 80 年代起成为关系数据库标准语言。

## 7. 关系完整性

关系模型有三类完整性约束：

1. 实体完整性
2. 参照完整性
3. 用户定义完整性

### 7.1 实体完整性

实体完整性规则：若属性 `A` 是基本关系 `R` 的主属性，则 `A` 不能取空值。

含义：

- 主码用于唯一标识元组。
- 主码的各个主属性不能为 NULL。
- 若主属性为空，就无法区分实体。

注意：实体完整性不仅约束单属性主码，也约束由多个属性组成的主码。

### 7.2 参照完整性

参照完整性描述关系与关系之间的引用约束。

若关系 `R` 的外码 `F` 参照关系 `S` 的主码 `Ks`，则 `R` 中 `F` 的取值必须满足：

- 要么为空值；
- 要么等于 `S` 中某个元组的主码值。

通俗理解：外码不能引用一个不存在的对象。

例子：

```text
学生选课(学号, 课程号, 成绩)
学生(学号, 姓名, 性别)
课程(课程号, 课程名, 学分)
```

`学生选课.学号` 必须对应 `学生.学号` 中已有的值；`学生选课.课程号` 必须对应 `课程.课程号` 中已有的值。

### 7.3 用户定义完整性

用户定义完整性是针对具体应用语义提出的约束。

例子：

- 成绩必须在 0 到 100 之间。
- 性别只能取“男”或“女”。
- 学分必须大于 0。
- 姓名不能为空。

## 8. 关系代数

关系代数是一种抽象的查询语言，用对关系的运算表达查询。

关系代数运算对象是关系，运算结果仍然是关系。

### 8.1 传统集合运算

传统集合运算包括：

- 并
- 差
- 交
- 笛卡尔积

设关系 `R` 和 `S` 具有相同的目 `n`，且对应属性来自同一个域。

#### 并

```text
R ∪ S = { t | t ∈ R 或 t ∈ S }
```

结果仍为 `n` 目关系，由属于 `R` 或属于 `S` 的元组组成。

#### 差

```text
R - S = { t | t ∈ R 且 t ∉ S }
```

结果由属于 `R` 但不属于 `S` 的元组组成。

#### 交

```text
R ∩ S = { t | t ∈ R 且 t ∈ S }
```

交可以用差表示：

```text
R ∩ S = R - (R - S)
```

#### 笛卡尔积

若 `R` 为 `n` 目关系，有 `k1` 个元组；`S` 为 `m` 目关系，有 `k2` 个元组，则：

- `R x S` 是 `n + m` 目关系。
- 元组个数为 `k1 x k2`。

### 8.2 专门的关系运算

专门的关系运算包括：

- 选择
- 投影
- 连接
- 除

#### 选择

选择是从关系中选出满足条件的元组。

记法：

```text
σF(R)
```

含义：从关系 `R` 中选择满足条件 `F` 的元组。

对应 SQL 思维：`WHERE` 过滤行。

#### 投影

投影是从关系中选出若干属性列，并去掉重复元组。

记法：

```text
πA(R)
```

含义：从关系 `R` 中取属性列 `A`。

对应 SQL 思维：`SELECT` 指定列。

#### 连接

连接是从两个关系的笛卡尔积中选出满足条件的元组。

一般形式：

```text
R ⋈ S
```

常见连接：

- 等值连接：连接条件使用 `=`
- 自然连接：特殊的等值连接，会比较同名属性，并在结果中去掉重复属性列

自然连接要点：

- 要求两个关系有同名属性。
- 同名属性值相等的元组才能连接。
- 结果中同名属性只保留一列。

#### 除

除运算用于表达“全部”“至少包含所有”等查询。

直观理解：

```text
R ÷ S
```

返回那些在 `R` 中能够与 `S` 的所有元组都匹配的属性值。

典型题型：

- 查询选修了全部课程的学生。
- 查询供应了某工程所需全部零件的供应商。
- 查询至少选修了某几个指定课程的学生。

除运算是本章较难点，复习时应重点结合例题理解。

## 9. 易混概念辨析

### 9.1 关系与关系模式

| 概念 | 含义 | 类比 |
| --- | --- | --- |
| 关系模式 | 对关系结构的描述 | 表结构 |
| 关系 | 某一时刻的元组集合 | 表中的数据 |

关系模式是型，关系是值。

### 9.2 候选码、主码、外码

| 概念 | 作用 |
| --- | --- |
| 候选码 | 能唯一标识元组的最小属性组 |
| 主码 | 从候选码中选定的一个 |
| 外码 | 引用其他关系主码的属性或属性组 |

### 9.3 主属性与非主属性

| 概念 | 判断标准 |
| --- | --- |
| 主属性 | 属于某个候选码 |
| 非主属性 | 不属于任何候选码 |

### 9.4 选择、投影、连接

| 运算 | 操作对象 | 作用 |
| --- | --- | --- |
| 选择 | 行 | 选出满足条件的元组 |
| 投影 | 列 | 选出指定属性列 |
| 连接 | 两个关系 | 按条件组合元组 |

### 9.5 实体完整性与参照完整性

| 完整性 | 约束对象 | 关键规则 |
| --- | --- | --- |
| 实体完整性 | 主码/主属性 | 主属性不能取空值 |
| 参照完整性 | 外码 | 外码值必须为空或引用已存在主码 |
