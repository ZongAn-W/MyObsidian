### 4.1 函数依赖的定义

设关系模式 `R(U)`，`X` 和 `Y` 是 `U` 的子集。如果在 `R` 的任一可能关系中，任意两个元组只要在 `X` 上的值相等，则在 `Y` 上的值也相等，则称：

```text
X -> Y
```

即 `X` 函数确定 `Y`，或 `Y` 函数依赖于 `X`。

通俗理解：

```text
X 的值一确定，Y 的值也唯一确定。
```

例子：

```text
Sno -> Sname
Sno -> School
School -> Mname
(Sno, Cno) -> Grade
```

### 4.2 函数依赖是语义概念

函数依赖不是只看某一张当前表的数据，而是由现实语义决定。

例如：

- 如果规定学号唯一标识学生，则 `Sno -> Sname` 成立。
- 如果允许同名学生，则 `Sname -> Sno` 不成立。
- 如果规定一个学院只有一名院长，则 `School -> Mname` 成立。

考试判断函数依赖时，要先看题目给出的语义约束。

### 4.3 平凡与非平凡函数依赖

| 类型 | 条件 | 例子 |
| --- | --- | --- |
| 平凡函数依赖 | `Y` 是 `X` 的子集 | `(Sno, Cno) -> Sno` |
| 非平凡函数依赖 | `Y` 不是 `X` 的子集 | `(Sno, Cno) -> Grade` |

一般讨论规范化时，重点关注非平凡函数依赖。

### 4.4 完全函数依赖与部分函数依赖

若 `X -> Y`，并且对 `X` 的任意真子集 `X'`，都有 `X'` 不能决定 `Y`，则称 `Y` 完全函数依赖于 `X`。

记作：

```text
X -F-> Y
```

若 `X -> Y`，但 `X` 的某个真子集也能决定 `Y`，则称 `Y` 部分函数依赖于 `X`。

记作：

```text
X -P-> Y
```

例子：

```text
(Sno, Cno) -> Grade      完全函数依赖
(Sno, Cno) -> School     部分函数依赖，因为 Sno -> School
```

### 4.5 传递函数依赖

若：

```text
X -> Y
Y -> Z
Y 不能决定 X
Z 不是 Y 的子集
```

则称 `Z` 传递函数依赖于 `X`。

记作：

```text
X -T-> Z
```

例子：

```text
Sno -> School
School -> Mname
所以 Sno -T-> Mname
```

传递依赖通常会导致更新异常和数据冗余。
