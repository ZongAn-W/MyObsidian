### 3.1 实体完整性的含义

实体完整性要求基本关系的主属性不能取空值，主码必须唯一标识元组。

关系模型中，实体完整性通常通过 `PRIMARY KEY` 定义。

### 3.2 单属性主码

列级定义：

```sql
CREATE TABLE Student (
    Sno CHAR(8) PRIMARY KEY,
    Sname VARCHAR(20),
    Ssex CHAR(1)
);
```

表级定义：

```sql
CREATE TABLE Student (
    Sno CHAR(8),
    Sname VARCHAR(20),
    Ssex CHAR(1),
    PRIMARY KEY (Sno)
);
```

### 3.3 多属性主码

多属性主码只能用表级约束定义。

```sql
CREATE TABLE SC (
    Sno CHAR(8),
    Cno CHAR(5),
    Grade SMALLINT,
    Semester CHAR(5),
    Teachingclass CHAR(8),
    PRIMARY KEY (Sno, Cno)
);
```

### 3.4 实体完整性检查

DBMS 检查：

- 主码值是否为空。
- 主码值是否唯一。

若插入或修改后的主码值为空，或与已有主码重复，则操作被拒绝。

常见实现：DBMS 通常会在主码上自动建立索引，以提高唯一性检查效率。
