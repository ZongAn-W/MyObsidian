并发操作可能导致四类典型问题：

1. 丢失修改
2. 脏读
3. 不可重复读
4. 幻读

### 3.1 丢失修改

丢失修改指两个事务读入同一数据并分别修改，==后写入的结果覆盖了先写入的结果==。

例：

```text
T1 读取 A = 16
T2 读取 A = 16
T1 修改 A = 15 并写回
T2 修改 A = 15 并写回
```

如果业务上两次售票都应减少余额，最终却只减少一次，就发生了丢失修改。

本质：

```text
一个事务的更新被另一个事务覆盖。
```

### 3.2 脏读

脏读指一个事务读到了另一个事务尚未提交的数据，而后者后来回滚。

例：

```text
T1 修改 C = 200，但未提交
T2 读取 C = 200
T1 回滚，C 恢复为 100
```

此时 T2 读到的 `C=200` 就是脏数据。

本质：

```text
读到了未提交事务的临时修改。
```

### 3.3 不可重复读

不可重复读指同一事务两次读取同一数据，期间另一个事务修改并提交，导致两次读取结果不同。

例：

```text
T1 第一次读 B = 100
T2 修改 B = 200 并提交
T1 第二次读 B = 200
```

本质：

```text
同一行数据两次读取值不同。
```

### 3.4 幻读

幻读也叫幻影现象。

含义：一个事务按某条件读取一组记录后，另一个事务插入或删除了满足该条件的记录，导致前一个事务再次按同条件读取时记录集合发生变化。

两种常见情况：

- 其他事务插入新记录，第二次查询多出记录；
- 其他事务删除记录，第二次查询少了记录。

本质：

```text
同一查询条件两次读取，记录集合数量变了。
```

### 3.5 四类问题对比

| 问题 | 关注对象 | 典型原因 | 关键词 |
|---|---|---|---|
| 丢失修改 | 写-写冲突 | 后写覆盖前写 | 覆盖 |
| 脏读 | 未提交数据 | 读到别人未提交修改 | 未提交 |
| 不可重复读 | 同一行 | 两次读取同一行值不同 | 行值变化 |
| 幻读 | 一组行 | 两次按条件读取集合不同 | 行数变化 |
