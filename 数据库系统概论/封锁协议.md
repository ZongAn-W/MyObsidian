封锁协议规定：

- 何时申请锁；
- 申请什么锁；
- 何时释放锁。

不同封锁协议能保证不同程度的数据一致性。

### 6.1 一级封锁协议

一级封锁协议：事务在修改数据前必须先加 X 锁，直到事务结束才释放。

作用：

- 防止丢失修改；
- 保证事务可恢复。

不足：

- 不要求读数据前加 S 锁；
- 不能防止脏读；
- 不能防止不可重复读。

### 6.2 二级封锁协议

二级封锁协议：在一级封锁协议基础上，事务读数据前必须先加 S 锁，读完即可释放。

作用：

- 防止丢失修改；
- 防止脏读。

不足：

- S 锁读完就释放；
- 同一事务后续再读时，其他事务可能已经修改；
- 不能保证可重复读。

### 6.3 三级封锁协议

三级封锁协议：在一级封锁协议基础上，事务读数据前必须先加 S 锁，直到事务结束才释放。

作用：

- 防止丢失修改；
- 防止脏读；
- 防止不可重复读。

与二级协议区别：

```text
二级：S 锁读完释放
三级：S 锁事务结束才释放
```

### 6.4 三种封锁协议对比

| 协议 | X 锁释放时间 | S 锁释放时间 | 防止丢失修改 | 防止脏读 | 防止不可重复读 |
|---|---|---|---|---|---|
| 一级 | 事务结束 | 不要求 S 锁 | 是 | 否 | 否 |
| 二级 | 事务结束 | 读完释放 | 是 | 是 | 否 |
| 三级 | 事务结束 | 事务结束 | 是 | 是 | 是 |

记忆：

```text
一级防丢失修改
二级防脏读
三级防不可重复读
```
