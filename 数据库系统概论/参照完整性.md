### 4.1 参照完整性的含义

参照完整性描述关系与关系之间的引用约束。

若关系 `R` 的外码 `F` 参照关系 `S` 的主码 `Ks`，则 `R` 中 `F` 的值必须满足：

- 等于 `S` 中某个元组的主码值；
- 或者取空值。

通俗理解：外码不能引用不存在的对象。

### 4.2 参照完整性的定义

```sql
FOREIGN KEY (<外码列>)
REFERENCES <被参照表>(<主码列>)
```

例子：

```sql
CREATE TABLE SC (
    Sno CHAR(8),
    Cno CHAR(5),
    Grade SMALLINT,
    Semester CHAR(5),
    Teachingclass CHAR(8),
    PRIMARY KEY (Sno, Cno),
    FOREIGN KEY (Sno) REFERENCES Student(Sno),
    FOREIGN KEY (Cno) REFERENCES Course(Cno)
);
```

### 4.3 参照完整性的违约处理

当删除或修改被参照表中的元组，导致参照关系不一致时，可以采用不同策略。

常见策略：

| 策略 | 含义 |
| --- | --- |
| NO ACTION | 拒绝执行该操作 |
| CASCADE | 级联执行相关操作 |
| SET NULL | 将参照关系中的外码值设为空 |
| SET DEFAULT | 将外码值设为默认值 |

示例：

```sql
FOREIGN KEY (Sno) REFERENCES Student(Sno)
ON DELETE CASCADE
ON UPDATE CASCADE
```

含义：

- 删除 Student 中某学生时，级联删除 SC 中该学生的选课记录。
- 修改 Student 中该学生学号时，级联修改 SC 中对应学号。

另一个例子：

```sql
FOREIGN KEY (Cno) REFERENCES Course(Cno)
ON DELETE NO ACTION
ON UPDATE CASCADE
```

含义：

- 若 SC 中仍有学生选了某课程，则不允许删除 Course 中该课程。
- 若 Course 中课程号更新，则级联更新 SC 中课程号。
