# 第八章 数据库编程：重要知识点总结

> 来源：《数据库系统概论（第6版）》第8章。重点掌握 SQL 表达能力的限制、扩展 SQL、WITH/WITH RECURSIVE、过程化 SQL、游标、存储过程、存储函数、JDBC 编程流程和 MVC 框架。

## 1. 本章学习主线

SQL 是一种高度非过程化语言，适合描述“要什么数据”，但在复杂业务逻辑、流程控制和应用交互方面有不足。

本章主要回答两个问题：

1. 如何扩展 SQL 本身的表达能力？
2. 如何在高级程序设计语言中使用数据库？

核心主线：

```text
SQL 表达能力有限
    -> 扩展 SQL：WITH、WITH RECURSIVE、内置函数
    -> 过程化 SQL：变量、流程控制、游标、异常、存储过程/函数
    -> 高级语言访问数据库：ODBC/JDBC
    -> 应用开发组织：MVC、DAO、ORM
```

## 2. SQL 表达能力的限制

SQL 面向集合，具有非过程化特点，适合查询、插入、删除、修改等基本操作。

但在以下场景中，仅使用普通 SQL 会比较困难：

- 递归查询，如查询课程的所有直接和间接先修课。
- 复杂计算，如 GPA、等级转换、累计统计。
- 需要条件判断和循环控制的业务逻辑。
- 需要用户与系统交互的应用功能。
- 需要在程序中根据执行结果继续处理的场景。

因此需要两类扩展：

```text
扩展 SQL 功能
在高级语言中使用 SQL
```

## 3. 扩展 SQL 的功能

### 3.1 WITH 子句

`WITH` 子句用于定义命名的临时结果集，提高 SQL 的可读性和可维护性。

一般格式：

```sql
WITH RS1 [(目标列)] AS (
    SELECT ...
),
RS2 [(目标列)] AS (
    SELECT ...
)
SELECT ...
FROM RS1, RS2
WHERE ...;
```

特点：

- `RS1`、`RS2` 是临时结果集名。
- 只在当前 SQL 语句中有效。
- 可避免复杂子查询重复书写。
- 有助于把复杂查询拆成多个逻辑步骤。

### 3.2 WITH RECURSIVE 子句

`WITH RECURSIVE` 用于递归查询，适合处理层次结构或可递归关系。

典型结构：

```sql
WITH RECURSIVE RS AS (
    seed_query
    UNION [ALL]
    recursive_query
)
SELECT ...
FROM RS;
```

组成：

| 部分 | 作用 |
| --- | --- |
| 种子查询 | 产生初始结果集 |
| 递归查询 | 基于上一轮结果继续查找 |
| UNION/UNION ALL | 合并每轮结果 |

例子思路：查询某门课程所有先修课。

```text
第一层：找直接先修课
第二层：找直接先修课的先修课
不断重复，直到没有新结果
```

易混点：

- `WITH` 是普通临时结果集。
- `WITH RECURSIVE` 适合递归层次关系。

### 3.3 引入新的内置函数

DBMS 通常提供丰富的内置函数，扩展 SQL 表达能力。

常见类型：

| 类型 | 例子 |
| --- | --- |
| 数学函数 | 绝对值、四舍五入 |
| 聚合函数 | 求和、平均值、最大值、最小值 |
| 字符串函数 | 长度、拼接、截取 |
| 日期时间函数 | 当前日期、日期加减 |
| 格式化函数 | 类型转换、格式化输出 |
| 控制流函数 | 条件判断 |
| 加密函数 | 字符串加密 |
| 系统信息函数 | 当前数据库、版本信息 |

内置函数适合完成常见计算，但复杂业务仍可能需要过程化 SQL 或高级语言。

## 4. 高级语言实现复杂应用

关系数据库管理系统和应用程序是两个相对独立的系统，需要通过接口传递数据和控制信息。

常见方式：

| 方式 | 含义 |
| --- | --- |
| 动态链接库调用 | 应用程序调用 DBMS 提供的库函数 |
| 嵌入式 SQL | 把 SQL 语句嵌入 C、Java 等主语言 |
| ODBC/JDBC | 通过标准接口访问数据库 |

### 4.1 嵌入式 SQL 的通信机制

嵌入式 SQL 中，SQL 与主语言之间需要通信。

主要机制：

| 机制 | 作用 |
| --- | --- |
| SQL 通信区 SQLCA | 向主语言返回 SQL 执行状态 |
| 主变量 | 主语言和 SQL 之间传递数据 |
| 指示变量 | 指示主变量是否为空值等状态 |
| 游标 | 处理多行查询结果 |

主变量通常需要在声明区中声明，并在 SQL 中用特殊标记引用。

### 4.2 ODBC/JDBC 中间件方式

ODBC/JDBC 提供统一的数据库访问接口，使应用程序可以用相似方式访问不同 DBMS。

优点：

- 提高应用程序可移植性。
- 屏蔽不同 DBMS 接口差异。
- 方便应用程序与数据库之间交互。

ODBC 更偏通用开放数据库连接；JDBC 是面向 Java 的数据库连接标准。

## 5. 过程化 SQL

过程化 SQL 是对基本 SQL 的扩展，在 SQL 基础上增加变量、流程控制、异常处理、游标、存储过程等机制。

常见产品：

- Oracle：PL/SQL。
- SQL Server：Transact-SQL。
- DB2：SQL PL。
- Kingbase：PL/SQL。

### 5.1 过程化 SQL 块结构

过程化 SQL 的基本单位是块。

典型结构：

```sql
DECLARE
    -- 定义变量、常量、游标、异常
BEGIN
    -- SQL 语句和流程控制语句
EXCEPTION
    -- 异常处理
END;
```

理解：

- `DECLARE`：定义部分。
- `BEGIN ... END`：执行部分。
- `EXCEPTION`：异常处理部分。

### 5.2 变量和常量

变量定义格式：

```sql
变量名 数据类型 [NOT NULL] [:= 初值表达式];
```

常量定义格式：

```sql
常量名 数据类型 CONSTANT := 常量表达式;
```

赋值语句：

```sql
变量名 := 表达式;
```

常量一旦赋值，在作用域内不能修改。

### 5.3 流程控制

过程化 SQL 支持条件控制和循环控制。

条件控制：

```sql
IF condition THEN
    statements;
END IF;
```

```sql
IF condition THEN
    statements1;
ELSE
    statements2;
END IF;
```

循环控制：

```sql
LOOP
    statements;
END LOOP;
```

```sql
WHILE condition LOOP
    statements;
END LOOP;
```

```sql
FOR count IN [REVERSE] bound1 .. bound2 LOOP
    statements;
END LOOP;
```

常见退出语句：

```sql
EXIT
BREAK
LEAVE
```

具体关键字因 DBMS 产品不同而略有差异。

### 5.4 异常处理

当过程化 SQL 执行时出现异常，应在异常处理部分捕获并处理。

作用：

- 避免程序异常终止。
- 根据错误类型执行补救措施。
- 记录或返回错误信息。

嵌入式 SQL 中常见执行状态信息放在 SQLCA 中；过程化 SQL 中通常有更完整的异常处理机制。

## 6. 游标

### 6.1 为什么需要游标

SQL 面向集合，一条查询可能返回多行；而高级语言或过程化逻辑通常一次处理一行。

游标用于在多行结果集中逐行移动和处理记录。

理解：

```text
游标 = 指向查询结果集中当前行的指针
```

### 6.2 游标的使用步骤

使用游标通常分 4 步：

1. 声明游标。
2. 打开游标。
3. 读取游标。
4. 关闭游标。

基本形式：

```sql
DECLARE 游标名 CURSOR FOR
SELECT ...;
```

```sql
OPEN 游标名;
```

```sql
FETCH 游标名 INTO 变量1, 变量2, ...;
```

```sql
CLOSE 游标名;
```

要点：

- 声明游标时只定义查询，不执行查询。
- 打开游标时执行查询，把结果放入缓冲区。
- `FETCH` 每次推进并取出一行。
- 关闭游标释放结果集资源。

### 6.3 游标适用场景

适合：

- 需要逐行处理查询结果。
- 每行处理逻辑不同或较复杂。
- 需要在循环中结合条件判断。

不适合：

- 能用集合 SQL 一次完成的操作。
- 大量数据逐行处理但没有必要逻辑差异的操作。

集合操作通常比逐行游标更高效。

## 7. 存储过程和存储函数

### 7.1 存储过程

存储过程是命名、编译并存储在数据库中的过程化 SQL 程序，供应用程序或其他过程调用。

创建格式：

```sql
CREATE OR REPLACE PROCEDURE 过程名 (
    [IN | OUT | INOUT] 参数名 数据类型,
    ...
)
AS
    过程化 SQL 块;
```

参数模式：

| 模式 | 含义 |
| --- | --- |
| IN | 输入参数 |
| OUT | 输出参数 |
| INOUT | 既输入又输出 |

调用格式：

```sql
CALL 过程名(参数1, 参数2, ...);
```

或：

```sql
PERFORM 过程名(参数1, 参数2, ...);
```

具体语法由 DBMS 决定。

### 7.2 修改和删除存储过程

修改名称：

```sql
ALTER PROCEDURE 过程名1 RENAME TO 过程名2;
```

重新编译：

```sql
ALTER PROCEDURE 过程名 COMPILE;
```

删除：

```sql
DROP PROCEDURE 过程名;
```

### 7.3 存储过程的优点

| 优点 | 说明 |
| --- | --- |
| 执行效率较高 | 预编译并存储在数据库服务器端 |
| 减少网络通信 | 客户端只需传过程名和参数 |
| 集中控制业务逻辑 | 逻辑放在数据库服务器统一维护 |
| 安全性较好 | 可通过授权调用过程而不直接授权访问表 |

### 7.4 存储函数

存储函数类似存储过程，但必须返回一个值。

创建格式：

```sql
CREATE OR REPLACE FUNCTION 函数名 (
    参数名 数据类型,
    ...
)
RETURNS 类型
AS
    过程化 SQL 块;
```

调用方式：

```sql
SELECT 函数名(参数1, 参数2);
```

区别：

| 概念 | 特点 |
| --- | --- |
| 存储过程 | 偏向执行操作，可有多个 IN/OUT 参数 |
| 存储函数 | 必须返回一个值，可在表达式或 SELECT 中使用 |

## 8. JDBC 编程

### 8.1 JDBC 的含义

JDBC 是 Java Database Connectivity，即 Java 数据库连接。

它提供 Java 程序访问数据库的标准接口，使 Java 应用能通过统一 API 操作不同数据库。

### 8.2 JDBC 应用系统结构

JDBC 结构通常包括：

```text
用户应用程序
    -> JDBC APIs
    -> JDBC 驱动程序管理器
    -> JDBC 驱动程序
    -> 数据源
```

JDBC 的作用：

- 提供统一数据库访问接口。
- 通过驱动程序连接具体 DBMS。
- 将 SQL 执行结果返回给 Java 程序。

### 8.3 JDBC 常用类和接口

| 类/接口 | 作用 |
| --- | --- |
| `Driver` | 数据库驱动程序类 |
| `DriverManager` | 管理驱动程序，建立连接 |
| `Connection` | 表示与数据库的连接 |
| `Statement` | 执行静态 SQL |
| `PreparedStatement` | 执行预编译、带参数 SQL |
| `CallableStatement` | 调用存储过程 |
| `ResultSet` | 处理查询结果集 |

### 8.4 JDBC 数据类型

JDBC 涉及两类数据类型：

- SQL 数据类型：数据库中的数据类型。
- Java 数据类型：应用程序变量的数据类型。

JDBC 负责在二者之间进行映射和转换。

### 8.5 JDBC 操作数据库的流程

典型流程：

```text
1. 加载驱动程序
2. 定义数据库连接 URL
3. 建立数据库连接
4. 创建语句执行对象
5. 执行 SQL 语句
6. 处理结果集
7. 关闭结果集、语句对象和连接，释放资源
```

### 8.6 加载驱动程序

传统方式：

```java
Class.forName("驱动程序类名");
```

JDBC 4.0 以后通常可自动加载驱动，但理解该步骤仍很重要。

### 8.7 建立连接

连接通常通过 `DriverManager.getConnection` 建立：

```java
Connection conn = DriverManager.getConnection(url, username, password);
```

URL 通常包含：

```text
协议名、子协议名、服务器地址、端口号、数据库名
```

### 8.8 执行 SQL 语句

常见执行对象：

| 对象 | 用途 |
| --- | --- |
| `Statement` | 执行静态 SQL |
| `PreparedStatement` | 执行带参数 SQL，适合重复执行和防 SQL 注入 |
| `CallableStatement` | 执行存储过程 |

常用执行方法：

| 方法 | 用途 | 返回 |
| --- | --- | --- |
| `executeQuery` | 查询语句 | `ResultSet` |
| `executeUpdate` | 插入、删除、更新、DDL | 影响行数或状态 |
| `execute` | 任意 SQL 或存储过程 | `boolean` |

### 8.9 处理结果集

`ResultSet` 表示查询结果集。

常用思路：

```java
while (rs.next()) {
    // 读取当前行字段
}
```

常见读取方法：

```java
getString()
getInt()
getBoolean()
getFloat()
```

可按列序号或列名读取字段。

### 8.10 释放资源

操作完成后应关闭：

```text
ResultSet
Statement / PreparedStatement / CallableStatement
Connection
```

原因：

- 释放数据库连接。
- 释放结果集缓冲区。
- 避免资源泄漏。
- 提高数据库性能。

## 9. MVC 框架下的数据库应用开发

### 9.1 MVC 的含义

MVC 是 Model-View-Controller，即模型-视图-控制器。

| 组成 | 作用 |
| --- | --- |
| Model | 业务模型，处理业务逻辑和数据访问 |
| View | 用户结果呈现，负责界面显示 |
| Controller | 业务模型控制器，接收请求并调度模型 |

基本流程：

```text
用户请求
    -> Controller
    -> Model 访问业务逻辑和数据库
    -> View 展示结果
```

### 9.2 MVC 的优点

- 分离界面、控制和业务逻辑。
- 降低模块耦合。
- 便于维护和扩展。
- 提高代码复用性。
- 适合 Web 数据库应用开发。

### 9.3 DAO

DAO 是 Data Access Object，数据访问对象。

作用：

```text
把数据库的增、删、改、查操作封装到专门的类中。
```

业务模型需要访问数据库时，调用 DAO 方法，而不是在业务代码中直接写 JDBC 每一步。

好处：

- 隐藏数据库访问细节。
- 减少重复代码。
- 便于统一修改数据库访问逻辑。

### 9.4 ORM

ORM 是 Object-Relational Mapping，对象-关系映射。

作用：

```text
把关系数据库中的表和业务模型中的对象建立映射。
```

使用 ORM 后，程序可以更偏向操作对象，而不是直接写复杂 SQL。

常见思想：

- 表对应类。
- 元组对应对象。
- 属性对应对象字段。
- 表之间关系对应对象之间关系。

## 10. 易混概念辨析

### 10.1 WITH 与 WITH RECURSIVE

| 概念 | 作用 |
| --- | --- |
| WITH | 定义临时结果集，使复杂查询更清晰 |
| WITH RECURSIVE | 定义递归临时结果集，处理层次或递归关系 |

### 10.2 游标与结果集

| 概念 | 含义 |
| --- | --- |
| 结果集 | SQL 查询返回的多行数据 |
| 游标 | 指向结果集中当前行的处理指针 |

### 10.3 存储过程与存储函数

| 概念 | 是否必须返回值 | 典型用途 |
| --- | --- | --- |
| 存储过程 | 不一定 | 执行业务操作 |
| 存储函数 | 必须 | 计算并返回一个值 |

### 10.4 Statement 与 PreparedStatement

| 对象 | 适用场景 |
| --- | --- |
| `Statement` | 执行简单静态 SQL |
| `PreparedStatement` | 执行带参数、重复执行的 SQL |

`PreparedStatement` 的优点：

- 可预编译。
- 可设置动态参数。
- 有助于防止 SQL 注入。
- 多次执行时效率更高。

### 10.5 JDBC 与 MVC

| 概念 | 层次 |
| --- | --- |
| JDBC | Java 访问数据库的底层接口 |
| MVC | 组织 Web 应用结构的开发框架 |
| DAO | 封装数据库访问 |
| ORM | 封装对象和关系表之间的映射 |
