> 来源：《数据库系统概论（第6版）》第4章。重点掌握数据库安全风险、安全控制技术、授权与回收、角色、强制访问控制、审计和数据加密。

## 1. 本章学习主线

数据库安全性是指保护数据库，防止因非法使用造成数据泄露、篡改或破坏。

本章主要解决三个问题：

1. 数据库为什么会不安全？
2. DBMS 如何控制用户访问？
3. 如何通过授权、角色、审计、加密等机制提高安全性？

核心记忆：

```text
数据库安全性 = 身份鉴别 + 存取控制 + 审计 + 加密 + 其他保护
```

## 13. 易混概念辨析

### 13.1 安全性与完整性

| 概念 | 关注点 |
| --- | --- |
| 安全性 | 防止非法访问、泄露、篡改和破坏 |
| 完整性 | 保证数据正确、有效、相容 |

### 13.2 GRANT OPTION 与 ADMIN OPTION

| 选项 | 用于 | 含义 |
| --- | --- | --- |
| WITH GRANT OPTION | 对象权限 | 用户可继续授予该对象权限 |
| WITH ADMIN OPTION | 角色 | 用户或角色可继续授予该角色 |

### 13.3 自主存取控制与强制存取控制

| 类型 | 控制依据 | 特点 |
| --- | --- | --- |
| DAC | 用户授权 | 灵活，可传播权限 |
| MAC | 安全级别和强制规则 | 更严格，用户不能随意改变 |

### 13.4 审计与加密

| 技术 | 作用 |
| --- | --- |
| 审计 | 记录和追踪操作 |
| 加密 | 防止数据被直接读懂 |

## 15. 速记版

```text
数据库安全性：
防止非法使用导致泄露、篡改、破坏

不安全因素：
非授权访问、敏感数据泄露、安全环境脆弱

安全控制：
身份鉴别、存取控制、审计、加密

DAC：
GRANT / REVOKE

角色：
一组权限的集合

MAC：
主体、客体、安全级别，系统强制控制

视图：
隐藏不该看的行或列

审计：
记录操作，事后追踪

加密：
明文变密文，保护存储和传输
```
