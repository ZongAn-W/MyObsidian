- 考虑页面载入内存后是否被修改的问题，增加了修改位
- 再访问位同时为0的进程建优先淘汰没有修改过的页面
- 访问位A   修改位M

| 页面  | 访问位A | 修改位M | 页面情况      |
| --- | ---- | ---- | --------- |
| 1类  | 0    | 0    | 最佳淘汰页     |
| 2类  | 0    | 1    | 并不是很好的淘汰页 |
| 3类  | 1    | 0    | 有可能再次被访问  |
| 4类  | 1    | 1    | 很可能再次被访问  |
### 算法执行过程
- 指针从当前位置开始扫描循环队列，寻找第一类页面，第一次扫描不修改访问位A
- 如果第一轮没找到开始第二轮扫描，寻找第二类页面，第二轮扫描会将访问位置0
- 第三轮重复第一步，如果失败，必要时重复第二步
- 极端情况四轮一定能找到要被淘汰的页
