# OS的目标
### 方便性
- 为用户和应用程序提供便捷的使用接口，降低计算机系统的使用门槛
- 对普通用户，OS提供了GUI和CLI
- 对应用程序开发者，OS提供了丰富的[[系统调用]]接口
 
### 有效性
提高系统==资源利用率==
- OS通过合理的资源分配和调度策略，避免资源的闲置和浪费
提高系统的==吞吐量==
- 吞吐量指的是单位时间内系统完成的任务数量
- OS通过优化进程调度算法、协调IO设备和CPU的工作节奏，减少进程的等待时间，提高整体处理效率

### 可扩充性
- OS需要具备灵活的可扩充性来满足新的硬件设备、软件功能和用户需求
- OS的架构因采用模块化设计，个功能模块（进程管理、内存管理、设备驱动）相对独立，模块之间通过标准化的接口通信
- 当需要添加新功能时，只需编写对应的模块或驱动程序，通过接口集成到现有系统中

### 开放性
- OS应遵循国际标准和规范，支持不同厂家的硬件设备和软件产品之间的兼容和互操作
- 使得用户可以根据需求选择不同的硬件和软件组合，不必受限于单一厂商的生态


# OS的功能
### 用户与计算机硬件系统之间的接口
1. 命令方式：CDM、Shell
2. [[系统调用]]方式：应用程序和[[操作系统内核]]进行交互的接口
3. 图形/窗口方式：窗口、图标、菜单、按钮
### 计算机系统资源的管理者
1. 对于CPU资源：[[进程调度]]进程调度算法
2. 对于内存资源：通过内存的分配、回收、地址映射[[离散分配管理方式]]、[[连续分配管理方式]]
3. 对外存和文件资源管理：[[文件管理]]
4. 对于IO设备：[[设备管理]]
### 实现了对计算机资源的抽象
- OS通过抽象的方式，屏蔽了硬件资源的物理细节，为用户和应用程序提供了统一简洁的逻辑接口
