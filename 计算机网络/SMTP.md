## SMTP

SMTP 是应用层协议，用来实现用户代理到邮件服务器以及邮件服务器之间的邮件传输，基于 **[[TCP]] 实现，端口号为 25，使用客户/服务器模型**。SMTP 完成邮件传输经历：连接建立、邮件传输阶段和连接释放。

① 连接建立：建立好 [[TCP]] 连接后，服务方发回 220 Service ready（服务就绪）。然后 SMTP 客户向 SMTP 服务器发送 HELO 命令，附上发送方的主机名。

【注意】SMTP 不使用中间邮件服务器进行中转。

② 邮件传送：客户发 `MAIL FROM:<iscas@hit.edu.cn>` → 服务器发 250 OK（如果准备好接收）→ `RCPT TO:<收件人地址>` 指示收件人地址（多个收件人就多条命令）→ 服务器回 250 OK 或 550 No such user here（无此用户）→ 客户发 DATA 传送数据。

③ 连接释放：客户应发送 QUIT 命令 → 服务器回 221 服务关闭。


## MIME 多用途互联网邮件扩充

传统电子邮件如 SMTP 只能传输 7 位 ASCII 字符，这是不够的，MIME 可以将非 ASCII 数据转换为 ASCII 字符使用 SMTP 协议传输。

具体做法：在邮件首部增加 MIME 首部行，说明主题内容原本的数据类型以及采用的编码标准。
