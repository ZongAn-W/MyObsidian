# jmp指令
- 无条件转移指令，直接跳转到目标地址执行
- PC里面指示的是当前执行指令的地址，每条指令执行之后，自动指向下一条地址，执行控制流指令时，可修改PC值，程序中常用标签label来标记指令地址（更灵活）
# jcondition指令
条件转移指令，根据程序状态字 寄存器中的状态标志来决定是否转移
![[jcondition指令.jpg]]
- 数据的比较来自于 cmp/test指令
- cmp：执行减法不保存结果，设置标识位
- test：按位与运算不保存结果，更新标志位ZF
# call指令
- ==实现子程序调用==指令，call指令将下一条指令的地址（返回地址）压入栈，然后转移到label标记子函数的地址。
- 通常用函数名作为函数起始地址的标签，方便转移
# ret指令
- 实现子程序的返回，ret从栈帧中弹出call指令压入的返回地址，恢复PC继续执行
# loop指令
- 配合ecx寄存器实现循环（ecx自动减1，不为0时则循环）
mov ecx, 5
BODY:
...
...
loop BODY // ecx=ecx-1若ecx不为0则跳转到BODY继续执行
